# 🔐 Registration Code System - Backend Implementation

## Overview
This system prevents unauthorized registration of Level Advisers and Exam Officers by requiring one-time registration codes generated by Super Admins. Only holders of these codes can register as these privileged roles.

**Protected Roles:**
- ✅ **Level Adviser** - Manages a specific level, assigns class reps
- ✅ **Exam Officer** - Creates and manages exams/CBTs

**Open Roles:**
- 🎓 **Student** - Anyone can register
- 📋 **Class Rep** - Assigned by Level Adviser to students

---

## 🏗️ Architecture

### Models
- **RegistrationCode** - Stores unique codes with usage tracking and expiry (supports both level_adviser and exam_officer roles)
- **User** - Updated to accept `registrationCode` in registration payload for both roles

### Controllers
- **Auth Controller** - Updated register function to validate codes for both level_adviser and exam_officer
- **New Admin Functions** - Generate, list, and revoke codes for any role

### Routes
- **Admin-only routes** - `/registration-codes/*` endpoints for code management

## 📋 What Was Implemented

### 1. RegistrationCode Model (`src/models/RegistrationCode.model.ts`)

**Fields:**
```typescript
{
  code: string;              // Unique code (e.g., "LA-100-A7K9P")
  role: string;              // "level_adviser" or "exam_officer"
  faculty?: string;          // Full faculty name (optional)
  facultyId?: string;        // Faculty ID for filtering (optional)
  level?: string;            // "100", "200", etc. (for level_adviser)
  isUsed: boolean;           // Whether code has been used
  usedBy?: ObjectId;         // User ID who used this code
  usedAt?: Date;             // When the code was used
  expiresAt: Date;           // When code expires (auto-delete via TTL)
  createdBy: ObjectId;       // Admin who created this code
  createdAt: Date;
  updatedAt: Date;
}
```

### 2. Updated Register Function

**Changes:**
- Validates `registrationCode` for both level_adviser AND exam_officer roles
- Checks if code is unused, valid, and not expired
- Enforces faculty/level restrictions on codes (if set)
- Marks code as used after successful registration for either role

**New Logic:**
```
IF role === "level_adviser" OR role === "exam_officer" THEN
  IF registrationCode is missing THEN reject
  IF registrationCode is invalid/expired/used THEN reject
  IF code role doesn't match requested role THEN reject
  IF code has restrictions AND they don't match THEN reject
  
  CREATE user with appropriate role
  MARK code as used (isUsed = true, usedBy = userId, usedAt = now)
ELSE IF role === "student" OR role === "class_rep" THEN
  (no code required)
ELSE IF role === "super_admin" THEN
  reject (super_admin can only be created by existing admin)
```

## 🔄 Complete Registration Flows

### Level Adviser Registration Flow

1. **Admin Generates Code**
```
Super Admin → POST /registration-codes/generate
Response: ["LA-100-XYZAB", "LA-100-PQRST", ...]
```

2. **Share Code with Level Adviser**
```
Admin → Email/SMS to Level Adviser
"Your registration code: LA-100-XYZAB"
```

3. **Level Adviser Registers**
```
POST /register
{
  "fullName": "Dr. Usman",
  "email": "usman@uni.edu",
  "password": "secure123",
  "role": "level_adviser",
  "facultyId": "education",
  "level": "100",
  "registrationCode": "LA-100-XYZAB"
}
```

4. **Backend Validates**
```
1. Check code exists for level_adviser
2. Check code is unused
3. Check code is not expired
4. Check faculty matches (if restricted)
5. Check level matches (if restricted)
6. Create user as level_adviser
7. Mark code as used
8. Return success
```

5. **Level Adviser Can Now Login**
```
Level Adviser logs in → Access User Management
→ Can assign Class Reps for their level
```

### Exam Officer Registration Flow

1. **Admin Generates Code**
```
Super Admin → POST /registration-codes/generate
Response: ["EO-5A9K2P", "EO-7X3M1L", ...]
```

2. **Share Code with Exam Officer**
```
Admin → Email/SMS to Exam Officer
"Your registration code: EO-5A9K2P"
```

3. **Exam Officer Registers**
```
POST /register
{
  "fullName": "Prof. Okafor",
  "email": "jokafor@uni.edu",
  "password": "secure456",
  "role": "exam_officer",
  "facultyId": "science",
  "registrationCode": "EO-5A9K2P"
}
```

4. **Backend Validates**
```
1. Check code exists for exam_officer
2. Check code is unused
3. Check code is not expired
4. Check faculty matches (if restricted)
5. Create user as exam_officer
6. Mark code as used
7. Return success
```

5. **Exam Officer Can Now Login**
```
Exam Officer logs in → Access Exam Management
→ Can create/manage exams and CBTs
```

## 🔒 Security Features

### Code Generation
- ✅ Unique codes with crypto-random part
- ✅ Format: `{role}-{level}-{random}` for clarity
- ✅ Configurable expiry (default 30 days)
- ✅ Optional faculty/level restrictions

### Code Validation
- ✅ Case-insensitive code matching
- ✅ Expiry checking (auto-delete expired codes)
- ✅ Single-use enforcement (isUsed flag)
- ✅ Audit trail (tracks who used code when)

### Authorization
- ✅ Only Super Admins can generate codes
- ✅ Only Super Admins can view/revoke codes
- ✅ No codes visible to Level Advisers
- ✅ Students can't bypass code requirement

### Restrictions
- ✅ Codes can be restricted to specific faculty
- ✅ Codes can be restricted to specific level
- ✅ Only one Level Adviser per faculty+level
- ✅ Can't register without valid code

## 🔄 Code Lifecycle

```
1. GENERATED
   ├─ Created with isUsed = false
   ├─ Assigned expiry date
   └─ Can be revoked (deleted)

2. ACTIVE
   ├─ Ready to use
   ├─ Not yet expired
   └─ Still unused

3. USED
   ├─ Claimed by someone registering
   ├─ Marked with usedBy and usedAt
   └─ Cannot be used again

4. EXPIRED
   ├─ Past expiryAt date
   ├─ Auto-deleted by TTL index
   └─ Cannot be used
```

## 🛠️ Configuration

### Code Expiry
```typescript
// Default: 30 days
const expiryDays = 30;
const expiryDate = new Date();
expiryDate.setDate(expiryDate.getDate() + expiryDays);
```

**To modify:** Pass different `expiryDays` in generate request

### Code Format
```typescript
// Level Adviser format: LA-100-A7K9P
// LA = Level Adviser
// 100 = Level (100, 200, 300, 400, 500)
// A7K9P = Random 3-byte hex (6 chars)

// Exam Officer format: EO-5A9K2P
// EO = Exam Officer
// 5A9K2P = Random 3-byte hex (6 chars, no level)

const randomPart = crypto.randomBytes(3).toString('hex').toUpperCase();
const prefix = role === 'level_adviser' ? `LA-${level}` : 'EO';
const code = `${prefix}-${randomPart}`;
```

**To modify:** Update the format string in `generateRegistrationCode()`
